Install Heph locally.

Use uv tool for the public command, then open Heph and check the installed version. 

uv tool install heph@latest
heph
heph --version

If your shell cannot find heph after install, run uv tool update-shell, restart the terminal, then try heph --version again.

Prefer pip for this Python environment? pip install heph

A harness around your documents, not a hosted workspace.

Hephaion provides the guardrails around Heph: retrieval, citation checks, scoped memory, and model boundaries. 

heph armory course-notes
cp notes.pdf ~/.armories/course-notes/materials/
heph course-notes

An armory is a normal folder under ~/.armories. Put source files in its materials/ directory, open the armory by name, then ask questions against that local scope.

Use /materials to choose what retrieval can see and /evidence to inspect the chunks behind an answer.

Create an armory

Each project gets a normal folder under ~/.armories with source files, local memory, saved chats, traces, and retrieval state.

Ask with evidence

Heph retrieves from enabled materials before the model answers, then keeps citations and evidence inspectable.

Keep models swappable

Use /login and /models for hosted providers, local llama.cpp, or a custom OpenAI-compatible endpoint.

Use the SDK when the interface is yours.

The browser surface can talk to a local bridge at 127.0.0.1:8766. Deployed pages stay static; your armory stays on your machine. 

npm run heph:sdk-bridge -- --armory ~/.armories/exams --allow-origin https://hephaion.vercel.app
# then refresh this page and ask in the terminal
